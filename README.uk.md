# Ishnekittt Прогноз Погоди

Інтерактивний веб-застосунок для прогнозування погоди, побудований на стеку React 19 + Vite. Визначними особливостями проєкту є кастомний рушій рендерингу метеорологічних карт із частинками вітру, динамічні відеофони та **клієнтська математична модель (AI)** для розрахунку локального прогнозу погоди безпосередньо у браузері.

## 🚀 Ключові можливості

- **Інтерактивна SVG/Canvas мапа:** Відображення регіонів (D3.js) із накладанням симуляції потоків вітру в реальному часі.
- **Динамічне відеосередовище:** Тло застосунку плавно адаптується під поточні погодні умови обраного регіону.
- **Детальна аналітика:** Компоненти для візуалізації УФ-індексу, атмосферного тиску, вологості та фаз сонця з використанням кастомних SVG-графіків.
- **Локальний AI Прогноз:** Альтернативний режим прогнозування, що працює без сторонніх серверів на базі лінійної регресії.
- **Абсолютна оптимізація:** Використання GPU-прискорення для анімацій хмар та Offscreen Canvas для складного кліпінгу масок.

## 🧩 Архітектура та взаємодія компонентів

Застосунок наслідує сувору компонентну архітектуру з централізованим керуванням станом через **Redux Toolkit** та **RTK Query**.

### Схема потоку даних:

1. **Ініціалізація (App.jsx):** Рендериться анімований фон з геометричними хмарами та компонент карти `Map`.
2. **Взаємодія з мапою:** При кліку на полігон SVG, `d3-geo` обчислює центроїд регіону. Redux Dispatcher оновлює `selectedRegion` в `appSlice`.
3. **Реактивність шару даних:** RTK Query (`weatherApi`) реагує на зміну координат і запитує дані (поточні, погодинні, на 10 днів, а також архів за 5 днів) через API (Open-Meteo).
4. **Відображення короткого віджета:** Анімовано з'являється `BriefWidget`, показуючи поточну погоду з попереднім завантаженням (`prefetching`) відеофону.
5. **Глибокий аналіз (FullModal):** При відкритті детального модального вікна користувач отримує доступ до графіків і перемикача "AI Mode".

### Керування станом (Store)

- **`appSlice`**: Зберігає UI-стан (`selectedRegion`, `isFullModalOpen`) та закешовані результати обчислень AI-моделі (`aiCalculations`).
- **`redux-persist`**: Інтегрований з `sessionStorage` для збереження кешу RTK Query між перезавантаженнями сторінки, мінімізуючи мережеві запити.

## 🧠 Математична AI-модель прогнозування

Однією з найскладніших концепцій проєкту є файл `utils/aiForecastUtils.js`, який реалізує прогностичний алгоритм без участі бекенду.

**Як це працює:**
Алгоритм бере масив фактичної погоди за останні **120 годин** (5 днів) і будує прогноз на наступні **24 години** за двома ключовими факторами:

1. **Глобальний тренд (Лінійна регресія):**
   Бібліотека `regression` аналізує весь масив зі 120 точок для кожного параметра (температура, тиск тощо) і будує базову лінію тренду за методом найменших квадратів.

2. **Експоненційне згладжування добової сезонності:**
   Погода є циклічною (вдень тепло, вночі холодно). Лінійна регресія не може цього врахувати. Тому алгоритм шукає різницю між передбаченим трендом та фактичним значенням у **ту ж саму цифрову координату години** в минулі дні (24, 48, 72 години тому).
   Далі застосовується вагова функція: `weight = 1 / daysAgo`. Відхилення, зафіксоване вчора (вага 1), впливає на прогноз значно сильніше, ніж відхилення 5 днів тому (вага 0.2).

3. **Евристичний розрахунок похідних параметрів:**
   Ймовірність опадів та погодні коди (weather codes) обчислюються на основі похідних (падіння тиску, стрибки вологості, прогнозований обсяг дощу).

## ⚡ Техніки оптимізації

- **Offscreen Canvas Masking (WindOverlay.jsx):** Щоб частинки вітру не виходили за межі країни, використовується складний SVG-шлях (clipPath). Замість того, щоб викликати важку операцію `clip()` кожен кадр в основному потоці, маска країни **одноразово** рендериться у віртуальний Canvas (`maskCanvas`), а потім накладається через `globalCompositeOperation = "destination-in"`. Це вирішує проблему критичних падінь FPS на мобільних пристроях (особливо iOS).
- **Апаратне прискорення (App.module.css):** Хмари анімуються виключно через `transform: translate3d`, а їхня форма будується геометрично (без `box-shadow` і `filter: blur`), що дозволяє GPU кешувати шари (властивість `will-change`).
- **Ліниве завантаження:** Модальні вікна та віджети завантажуються через `React.lazy` та `Suspense`.

## 🛠 Технічний стек

- **Ядро:** React 19, Vite (React Compiler enabled)
- **State Management:** Redux Toolkit, RTK Query, Redux Persist
- **Математика та Карти:** d3-geo, regression, suncalc
- **Стилізація:** CSS Modules, clsx
