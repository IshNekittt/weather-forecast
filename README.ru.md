# Ishnekittt Weather Forecast

Интерактивное веб-приложение для прогнозирования погоды, построенное на стеке React 19 + Vite. Отличительными особенностями проекта являются кастомный движок рендеринга метеорологических карт с частицами ветра, динамические видеофоны и **клиентская математическая модель (AI)** для расчета локального прогноза погоды непосредственно в браузере.

## 🚀 Ключевые возможности

- **Интерактивная SVG/Canvas карта:** Отображение регионов (D3.js) с наложением симуляции потоков ветра в реальном времени.
- **Динамическое видео-окружение:** Фон приложения плавно адаптируется под текущие погодные условия выбранного региона.
- **Детальная аналитика:** Компоненты для визуализации УФ-индекса, атмосферного давления, влажности и фаз солнца с использованием кастомных SVG-графиков.
- **Локальный AI Прогноз:** Альтернативный режим прогнозирования, работающий без сторонних серверов на базе линейной регрессии.
- **Абсолютная оптимизация:** Использование GPU-ускорения для анимаций облаков и Offscreen Canvas для сложного клиппинга масок.

## 🧩 Архитектура и взаимодействие компонентов

Приложение следует строгой компонентной архитектуре с централизованным управлением состоянием через **Redux Toolkit** и **RTK Query**.

### Схема потока данных:

1. **Инициализация (App.jsx):** Рендерится анимированный фон с геометрическими облаками и компонент карты `Map`.
2. **Взаимодействие с картой:** При клике на полигон SVG, `d3-geo` вычисляет центроид региона. Redux Dispatcher обновляет `selectedRegion` в `appSlice`.
3. **Реактивность слоя данных:** RTK Query (`weatherApi`) реагирует на изменение координат и запрашивает данные (текущие, почасовые, на 10 дней, а также архив за 5 дней) через API (Open-Meteo).
4. **Отображение краткого виджета:** Анимированно появляется `BriefWidget`, показывая текущую погоду с предварительной загрузкой (`prefetching`) видеофона.
5. **Глубокий анализ (FullModal):** При открытии детального модального окна пользователь получает доступ к графикам и переключателю "AI Mode".

### Управление состоянием (Store)

- **`appSlice`**: Хранит UI-состояние (`selectedRegion`, `isFullModalOpen`) и закешированные результаты вычислений AI-модели (`aiCalculations`).
- **`redux-persist`**: Интегрирован с `sessionStorage` для сохранения кеша RTK Query между перезагрузками страницы, минимизируя сетевые запросы.

## 🧠 Математическая AI-модель прогнозирования

Одной из самых сложных концепций проекта является файл `utils/aiForecastUtils.js`, который реализует прогностический алгоритм без участия бэкенда.

**Как это работает:**
Алгоритм берет массив фактической погоды за последние **120 часов** (5 дней) и строит прогноз на следующие **24 часа** по двум ключевым факторам:

1. **Глобальный тренд (Линейная регрессия):**
   Библиотека `regression` анализирует весь массив из 120 точек для каждого параметра (температура, давление и т.д.) и строит базовую линию тренда по методу наименьших квадратов.

2. **Экспоненциальное сглаживание суточной сезонности:**
   Погода циклична (днем тепло, ночью холодно). Линейная регрессия не может это учесть. Поэтому алгоритм ищет разницу между предсказанным трендом и фактическим значением в **ту же самую цифровую координату часа** в прошлые дни (24, 48, 72 часа назад).
   Далее применяется весовая функция: `weight = 1 / daysAgo`. Отклонение, зафиксированное вчера (вес 1), влияет на прогноз значительно сильнее, чем отклонение 5 дней назад (вес 0.2).

3. **Эвристический расчет производных параметров:**
   Вероятность осадков и погодные коды (weather codes) вычисляются на основе производных (падение давления, скачки влажности, прогнозируемый объем дождя).

## ⚡ Техники оптимизации

- **Offscreen Canvas Masking (WindOverlay.jsx):** Чтобы частицы ветра не выходили за пределы страны, используется сложный SVG-путь (clipPath). Вместо того чтобы вызывать тяжелую операцию `clip()` каждый кадр в основном потоке, маска страны **единожды** рендерится в виртуальный Canvas (`maskCanvas`), а затем накладывается через `globalCompositeOperation = "destination-in"`. Это решает проблему критических падений FPS на мобильных устройствах (особенно iOS).
- **Аппаратное ускорение (App.module.css):** Облака анимируются исключительно через `transform: translate3d`, а их форма строится геометрически (без `box-shadow` и `filter: blur`), что позволяет GPU кэшировать слои (свойство `will-change`).
- **Ленивая загрузка:** Модальные окна и виджеты загружаются через `React.lazy` и `Suspense`.
